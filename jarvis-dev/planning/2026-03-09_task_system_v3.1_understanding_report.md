## 1. 개요 및 목적
벙커 팀(송PO)이 기획하고 김감사 QA팀이 승인한 **[업무 관리 v3.1]** 기획안을 바탕으로, 자비스 개발팀의 기술적 구현 방향을 정의합니다. 이번 업데이트의 핵심은 **"협업 책임성 강화(수락제)"**와 **"데이터 자산화(참고사항 히스토리)"**입니다.

## 2. 자비스 팀 기술 분석 (Technical Analysis)

### [BE] 👩‍💻 에이다 (Backend)
- **DB 확장**: `Tasks` 시트의 J, K열을 신규 할당하여 기존 인덱스 참조 코드를 보호합니다.
- **동시성 제어**: `LockService.getScriptLock()`을 활용해 프로젝트 생성 시 코드 중복을 0%로 차단하겠습니다.
- **보안 필터링**: `참고사항` 입력값에서 `=`로 시작하는 시트 수식과 `<script>` 태그를 정규표현식으로 사전 차단합니다.
- **알림 엔진**: 기존 `slack_notification.gs`를 확장하여 `Start_Date` 기반 D-3 멘션 알림 로직을 구현합니다.

### [FE] 👧 클로이 (Frontend)
- **인라인 UI**: 하위 레이어를 여는 대신 프로젝트 선택 셀렉트박스 내에서 입력 폼이 토글되는 인라인 방식을 적용하여 UX를 간소화합니다.
- **수락 버튼**: `수락대기` 상태 업무에 대해서만 버튼을 활성화하며, 클릭 시 즉시 상태가 동기화되도록 구현합니다.
- **히스토리 타임라인**: 업무 상세 보기 하단에 타임라인형 참고사항 섹션을 추가합니다.

### [TL] 👨‍💻 알렉스 (Tech Lead)
- **인덱스 전수 조사**: 기존 `ai_chat.gs` 등에서 `getRange()`나 배열 인덱스를 사용하는 모든 코드를 검토하여, 열 추가에 따른 영향도가 없는지 최종 확인하겠습니다.
