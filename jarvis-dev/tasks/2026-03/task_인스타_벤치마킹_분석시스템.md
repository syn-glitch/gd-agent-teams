## 지시 원문

> 퍼포먼스 에이전트 개발요청 문서를 분석해줘
> 인스타 계정은 생성 전이야, 먼저 모듈B 다른 사람의 인스타 채널 분석을 먼저 만들자
> task.md 만들어줘 일단 수동으로해서 빠르게 결과물 보고 자동화를 하자

## 에이전트 이해 요약

- **핵심 요청**: 벤치마킹 채널 데이터를 수동 입력하고, 자동 분석/비교 대시보드를 만들어라
- **전략**: Phase 1(수동 입력 + 자동 분석) → Phase 2(데이터 수집 자동화)
- **작업 범위**: 모듈 B만 선행 개발. 모듈 A(내 채널 API)는 인스타 계정 생성 후 별도 태스크
- **완료 기준**: 벤치마킹 채널 3~5개 데이터를 입력하면, 비교 대시보드가 자동으로 계산/갱신되는 구글 시트 완성

---

## Phase 1: 수동 입력 + 자동 분석 (MVP)

### 작업 단계

- [ ] **1단계: 구글 시트 생성 + 스키마 설정** (벙커/에이다)
  - [ ] 1-1. 새 구글 스프레드시트 생성 ("인스타 벤치마킹 분석 시스템")
  - [ ] 1-2. [시트 1] `Benchmark_Channels` — 벤치마킹 채널 목록
    - 컬럼: ChannelID(@핸들), ChannelName, Category, Followers, Bio, AddedAt, IsActive, Notes
  - [ ] 1-3. [시트 2] `Benchmark_Data` — 주간 수집 데이터 (수동 입력)
    - 컬럼: CollectedAt, ChannelID, Followers, PostCount_Week, AvgLikes, AvgComments, EstEngagementRate, TopHashtags, TopHookPatterns, ContentMix(릴스/피드/캐러셀 비율)
  - [ ] 1-4. [시트 3] `Content_Log` — 개별 게시물 기록 (선택)
    - 컬럼: CollectedAt, ChannelID, PostURL, PostType, Caption_FirstLine, Likes, Comments, Hashtags, PostedAt
  - [ ] 1-5. [시트 4] `Dashboard` — 자동 분석 대시보드
    - 컬럼: Date, ChannelID, ChannelName, Followers, FollowerGrowth(전주 대비), EngRate, EngRateChange, PostFreq, TopContent(가장 반응 좋은 게시물)
  - [ ] 1-6. [시트 5] `Summary` — 전체 비교 요약
    - 컬럼: ChannelID, ChannelName, LatestFollowers, AvgEngRate, AvgPostFreq, TopHashtags, HookPattern, 강점분석

- [ ] **2단계: GAS 자동 계산 함수 개발** (에이다 BE)
  - [ ] 2-1. `calculateEngagementRate()` — 참여율 자동 계산 ((좋아요+댓글)/팔로워)
  - [ ] 2-2. `calculateFollowerGrowth()` — 전주 대비 팔로워 증감 계산
  - [ ] 2-3. `updateDashboard()` — Benchmark_Data → Dashboard 시트 자동 갱신
  - [ ] 2-4. `updateSummary()` — 전체 채널 비교 요약 자동 갱신
  - [ ] 2-5. `onEdit()` 트리거 — 데이터 입력 시 대시보드 자동 갱신

- [ ] **3단계: 입력 편의 기능** (클로이 FE / 에이다 BE)
  - [ ] 3-1. 데이터 입력용 드롭다운 (채널 목록 자동 연동)
  - [ ] 3-2. 데이터 유효성 검증 (숫자 범위, 필수값 체크)
  - [ ] 3-3. 조건부 서식 (참여율 높으면 녹색, 낮으면 빨강)
  - [ ] 3-4. 차트 자동 생성 (팔로워 추이, 참여율 비교 바 차트)

- [ ] **4단계: 테스트 + 대표 검수**
  - [ ] 4-1. 벤치마킹 채널 3~5개 샘플 데이터 입력
  - [ ] 4-2. 대시보드 자동 계산 정상 동작 확인
  - [ ] 4-3. 차트/요약이 정확한지 검증
  - [ ] 4-4. 대표님 검수 → 피드백 반영
