## 에이전트 이해 요약

- **핵심 요청**: 주디 우측 사이드바에 실시간 질의응답이 가능한 채팅 인터페이스 및 통합 컨텍스트 백엔드 엔진 개발.
- **작업 범위**: 좁은 폭에 최적화된 블록 레이아웃 UI 개발, CacheService를 이용한 데이터 요약 파이프라인 개발, 보안 검증.
- **완료 기준**: 모바일에서도 동작하는 주디 채팅창을 통해 실제 시스템 내 통합 데이터 기반 AI 응답이 성공적으로 수행되고 렌더링됨.

## 작업 단계

- [ ] 단계 1: [FE] 하단 Sticky 채팅 텍스트 입력 UI 및 노션 스타일 블록 메시지 UI 추가 (`judy_workspace.html`)
- [ ] 단계 2: [BE] `buildAndCacheUserContext()` 함수 구현 및 정기/조회 시 캐싱 로직 구성 (`ai_briefing.gs` / `ai_chat.gs`)

- [ ] 단계 4: [FE] `google.script.run` 연동하여 채팅 전송, 로딩 표시, 스트리밍(또는 결과 반환 후) 마크다운 렌더링 구현
- [ ] 단계 5: [통합] 버그 점검, 타임아웃 테스트 완료 후 배포 제안
