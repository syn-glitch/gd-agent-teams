## 🛫 프리플라이트 체크리스트

### PHASE 0: 고통 리포트 사전 열람 (최우선)

- [ ] **PRE-0**: `고통_리포트/` 폴더의 기존 실패 사례를 읽고, 이번 작업에 동일 패턴이 없는지 확인했는가?
  - 동일 패턴 감지 시 → **즉시 작업 중단 + 대표님에게 보고**
  - ⚠️ 이 항목을 건너뛰면 나머지 체크리스트의 의미가 없음

### PHASE 1: 환경 준비 (착수 전)

- [ ] **ENV-1**: 이 작업은 기존 프로젝트 안에서 하는 건가, 완전히 새로운 프로젝트인가?
  - 기존 프로젝트 내 기능 추가 → 기존 폴더에서 진행 ✅
  - **새 프로젝트/서비스** → 반드시 **새 윈도우(세션) + 새 폴더** 생성 후 진행 ✅
  - ❌ 기존 프로젝트 폴더 안에서 다른 프로젝트 코드를 섞는 것은 **금지**
- [ ] **ENV-2**: 작업 폴더 경로를 대표님에게 보고하고 확인받았는가?
- [ ] **ENV-3**: `.clasp.json`, 배포 설정 등 환경 파일이 올바른 프로젝트를 가리키고 있는가?

### PHASE 2: 기획 검증 (설계 확정 전)

- [ ] **PLAN-1**: PRD 또는 기획서가 작성되었는가? (벙커팀 또는 담당팀)
- [ ] **PLAN-2**: 기획서에 대해 관련 팀(벙커, 자비스, 김감사, 꼼꼼이)의 의견을 수렴했는가?
- [ ] **PLAN-3**: **김감사 QA팀의 사전 검수를 통과했는가?**
  - ❌ QA 반려 상태에서 구현 착수 금지
  - ❌ "일단 만들면서 고치자"는 금지
- [ ] **PLAN-4**: 역할 분담이 명확한가? (어느 팀이 무엇을 담당하는지)

### PHASE 3: 실행 중 점검 (구현 중)

- [ ] **EXEC-1**: 자동 실행(auto-proceed) 모드를 끄고, 단계별로 대표님 확인을 받고 있는가?
  - 최소 3단계 중간 보고: **기획 완료 → 구현 중간 → 배포 전**
- [ ] **EXEC-2**: 중간 산출물을 `shared/` 폴더에 공유하여 김감사 병렬 검수(Side-car QA)를 지원하는가?
- [ ] **EXEC-3**: 같은 오류가 2회 이상 반복되면 **즉시 멈추고 대표님에게 현황 보고**했는가?

### PHASE 4: 배포 검증 (배포 전)

- [ ] **DEPLOY-1**: Pre-Push Secret Scan을 완료했는가? (API 키, 토큰 노출 여부)
- [ ] **DEPLOY-2**: 배포 명령(`clasp push`, `git push` 등)의 출력을 확인했는가?
  - ❌ 출력 없이 exit 0 = 성공이 아님. "Pushed N files" 메시지 필수 확인
- [ ] **DEPLOY-3**: **배포 후 실제 URL에 접속하여 눈으로 동작을 확인했는가?**
  - ❌ 터미널 결과만으로 "배포 완료" 보고 금지
  - ✅ 실제 화면 스크린샷 또는 데이터 확인 증거 첨부 필수
- [ ] **DEPLOY-4**: 배포 결과를 대표님에게 보고하고, 대표님이 직접 확인할 수 있는 링크를 제공했는가?
