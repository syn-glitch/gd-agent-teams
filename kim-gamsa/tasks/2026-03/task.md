## 📋 QA 개요
Judy Workspace v3.1.2 및 v3.1.3 배포에 따른 슬랙 알림 연동 및 워크플로우 안정성을 검증해 주세요. 특히 '보안' 측면에서의 토큰 노출 여부와 '링크'의 정상 동작 여부가 핵심입니다.

## 🧪 테스트 항목

### 1. 슬랙 DM 알림 & 링크 (v3.1.2)
- [ ] **업무 배정 알림**: `/노트` 명령어나 웹에서 업무 등록 시 담당자에게 DM이 오는지 확인.
- [ ] **수락 알림**: 담당자가 업무를 수락했을 때 신청자(대표님)에게 DM이 오는지 확인.
- [ ] **링크 유효성**: DM 내의 "워크스페이스 바로가기" 링크가 실제 웹 앱 주소로 연결되는지 확인.
- [ ] **직접 발송 방식**: `chat.postMessage` 직접 호출 방식 전환 후 발송 성공 여부(스코프 에러 발생 여부).

### 2. 업무 완료 알림 & UI 동기화 (v3.1.3)
- [ ] **완료 알림**: 담당자가 상태를 '완료'로 변경 시 신청자에게 DM이 발송되는지 확인.
- [ ] **UI 상태 동기화 (🚨 중요)**: 
    - 신청자(Requester) 화면에서 업무 상태가 '전달'로 고정되지 않고, 실제 진행 상태(수락대기/진행중/완료 등)가 실시간 반영되는지 확인.
    - 담당자가 수락/진행중/완료로 변경 시 신청자 대시보드에서 즉시 뱃지 및 카운트가 업데이트되는지 검수.

### 3. 보안 및 거버넌스 (꼼꼼이 검수 사항)
- [ ] **코드 내 시크릿**: `github_issue.gs` 등 모든 서버 코드 내에 하드코딩된 토큰이 없는지 재검수.
- [ ] **권한**: 토큰 사용량 대시보드(`tokens` 탭)가 허용된 사용자(송용남, 정혜림)에게만 보이는지 확인.

### 4. 기타 모아보기 기능
- [ ] **토큰 대시보드**: Google Charts 기반 데이터 시각화가 정상적으로 로드되는지 확인.
