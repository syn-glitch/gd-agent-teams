# 🛰️ 주디(Judy) 슬랙 연동 고도화 계획 (Roadmap)

## 1. 개요
주디 에이전트의 인지 범위를 기존의 정형 데이터(시트, 캘린더)를 넘어 실시간 소통 채널인 **슬랙(Slack)**으로 확장합니다. 이를 통해 대표님과 팀원들은 슬랙을 일일이 확인하지 않고도 주디를 통해 놓친 대화나 핵심 이슈를 즉시 파악할 수 있습니다.

## 2. 단계별 고도화 로드맵

### 🚀 Phase 1: 실시간 인지 및 조회 (2026년 3월 중순)
- **핵심 기능**: 사용자가 명시적으로 물어볼 때 슬랙 데이터를 가져와 답변
- **주요 태스크**:
    - `handleMentions`: "오늘 나 태그된 대화 보여줘" 요청 처리
    - `findUnreadThreads`: "내가 답변 안 한 스레드 있어?" 요청 처리
- **기술 스펙**: `conversations.history` 및 `users.info` API 연동

### 📊 Phase 2: 지능형 브리핑 및 요약 (2026년 3월 말)
- **핵심 기능**: 주디가 능동적으로 슬랙 이슈를 요약하여 브리핑에 포함
- **주요 태스크**:
    - **채널별 이슈 트래킹**: 주요 채널(#벙커, #강철 등)의 최근 대화 맥락 요약
    - **통합 출근 브리핑**: "출근" 입력 시 시트 데이터 + 슬랙 미답변 항목 통합 보고
- **기술 스펙**: LLM 기반 대화 맥락 요약 알고리즘 적용

### 🧠 Phase 3: 지식 자산화 (2026년 4월 이후)
- **핵심 기능**: 슬랙 내 의사결정 사항을 주디가 자동으로 마크다운 노트로 저장
- **주요 태스크**:
    - **Auto-Archiving**: 슬랙의 특정 이모지(예: 📝) 반응 시 해당 대화를 `Notes` 시트 또는 마크다운 문서로 자동 저장
    - **Team Memory**: 팀 전체가 공유해야 할 지식을 슬랙에서 추출하여 RAG 데이터로 변환
- **기술 스펙**: 슬랙 Event Subscriptions 및 벙커 박DC(문서) 협업

## 3. 기대 효과
1. **커뮤니케이션 오버헤드 감소**: 수많은 슬랙 메시지 중 본인에게 정말 중요한 정보만 필터링 가능.
2. **정보 유실 방지**: 실시간 대화 중 발생하는 의사결정 사항을 빠짐없이 기록 및 자산화.
3. **업무 몰입도 향상**: 슬랙 알림에 매몰되지 않고 주디를 통해 정리된 피드백만 확인하며 업무 집중.
