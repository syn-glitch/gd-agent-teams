## 지시 원문

> 에이전트 호출 시동어가 너무 길어서 매번 복사 붙여넣기해야 하고, 불러와도 매번 업무 룰 숙지 여부를 질문해야 하며, 터미널/비터미널/새 세션에서도 동일하게 작동해야 한다. 해결 방법을 알려줘. 자비스팀 대상으로 테스트 후 확대.

## 에이전트 이해 요약

- **핵심 요청**: 에이전트 호출을 간소화하고, 모든 환경/세션에서 규칙이 자동 적용되게 하라
- **작업 범위**: Claude Code 환경(터미널+VS Code) 해결 방안 설계 + Antigravity 환경 조사
- **완료 기준**: `/자비스` 한 마디로 자비스팀 팀장이 규칙 숙지 상태로 활성화되는 것

## 문제 분석

| # | 문제 | AS-IS | TO-BE |
|---|------|-------|-------|
| 1 | 시동어 과다 | 3줄 복붙 필요 | `/자비스` 한 마디 |
| 2 | 규칙 숙지 확인 | 매번 수동 질문 | 자동 로드 + 자동 확인 |
| 3 | 비터미널 환경 | 설정 미적용 | 동일 적용 |
| 4 | 새 세션 일관성 | 매번 재설정 | 자동 적용 |

## 환경별 조사 결과

### A. Claude Code (터미널 CLI + VS Code 확장)

**사용 가능한 메커니즘 2가지:**

| 메커니즘 | 위치 | 기능 | 세션 지속 |
|---------|------|------|----------|
| **CLAUDE.md** | 프로젝트 루트 | 모든 세션에 자동 로드되는 시스템 지시 | ✅ 영구 |
| **커스텀 슬래시 명령** | `.claude/commands/자비스.md` | `/자비스` 입력 시 해당 내용이 프롬프트로 삽입 | ✅ 영구 |

- CLAUDE.md: 터미널 CLI, VS Code 확장 **모두** 자동 로드
- .claude/commands/: 터미널 CLI, VS Code 확장 **모두** `/명령어` 지원
- 새 세션, 새 윈도우에서도 프로젝트 디렉토리 기준으로 자동 적용

### B. Antigravity Agent (Gemini 3 Flash)

**조사 결과:**
- 프로젝트 레벨 시스템 지시 파일: ❌ 미지원 (CLAUDE.md 같은 메커니즘 없음)
- 슬래시 명령 커스텀: ❌ 로컬 설정 불가 (서버 제공만)
- 설정 저장: SQLite DB (`state.vscdb`) + protobuf (로컬 편집 불가)
- brain/ 디렉토리: `~/.gemini/antigravity/brain/` (대화 메모리, 프로젝트 지시용 아님)

## 해결 방안 제안

### 방안 1: CLAUDE.md + 커스텀 슬래시 명령 (Claude Code 전용) — ✅ 권장

**구조:**

```
프로젝트 루트/
├── CLAUDE.md                          ← 모든 세션 자동 로드 (공통 규칙 요약)
└── .claude/
    └── commands/
        ├── 자비스.md                   ← /자비스 → 자비스 팀장 활성화
        ├── 김감사.md                   ← /김감사 → 김감사 팀장 활성화
        ├── 꼼꼼이.md                   ← /꼼꼼이 → 꼼꼼이 팀장 활성화
        ├── 강철.md                     ← /강철 → 강철 팀장 활성화
        └── 벙커.md                     ← /벙커 → 벙커 팀장 활성화
```

**동작 흐름:**
1. 새 세션 시작 → CLAUDE.md 자동 로드 (공통 규칙 인지)
2. 대표가 `/자비스` 입력
3. `.claude/commands/자비스.md` 내용이 프롬프트로 삽입
4. Claude가 COMMON_RULES + jarvis-dev_DESIGN.md 읽고 자비스 팀장으로 활성화
5. 규칙 숙지 상태로 즉시 업무 대기

**CLAUDE.md 내용 (안):**
- 프로젝트 개요 (간략)
- 5개 팀 구조 (팀명, 역할, 호출 명령어)
- 공통 규칙 핵심 요약 (전문은 COMMON_RULES.md 참조)
- "에이전트 호출 시 반드시 해당 DESIGN.md를 읽고 역할을 활성화하라"

**슬래시 명령 파일 내용 (안) — 자비스.md:**
```
GD_Agent_teams/COMMON_RULES_local_backup.md 와
GD_Agent_teams/jarvis-dev/jarvis-dev_DESIGN.md 를 읽고
자비스 팀장으로 활동해줘.
규칙 숙지 완료 후 역할 활성화 확인 보고를 출력하라.
```

### 방안 2: Antigravity 환경 — ⚠️ 제한적 대안

Antigravity는 프로젝트 레벨 시스템 지시를 지원하지 않으므로:

| 대안 | 실행 가능성 | 한계 |
|------|-----------|------|
| A. Antigravity "System Instructions" UI 설정 | ⚠️ 조사 필요 | 프로젝트별 분리 불가능할 수 있음 |
| B. 대화 시작 시 첫 메시지로 규칙 파일 참조 | ✅ 가능 | 매번 1회는 필요 (단, 복붙보다 짧게 가능) |
| C. Antigravity의 brain/ 메모리에 학습시키기 | ⚠️ 비공식 | 안정성 보장 불가 |

**현실적 제안**: Claude Code에서 `/자비스` 사용을 권장하고, Antigravity는 보조 도구로 활용
