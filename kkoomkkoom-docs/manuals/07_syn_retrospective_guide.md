# 📝 07. SYN 전용 회고록 작성 가이드 (Retrospective Guide)

본 문서는 대표님(SYN)의 개인적 통찰과 시스템 운영 소회를 기록하는 **'SYN_회고'** 업무의 표준 절차를 정의합니다. 꼼꼼이 문서팀의 모든 팀원은 본 가이드를 준수하여 회고록을 관리해야 합니다.

---

## 1. 목적
- 대표님의 성장을 기록하고 아카이빙함.
- 에이전트 시스템 운영의 병목을 파악하고 차기 전략에 반영함.
- 대표님의 '원문'을 보존하여 의사결정의 컨텍스트를 유지함.

## 2. 저장소 및 파일 규격
- **저장 위치**: `GD_Agent_teams/SYN_회고/`
- **파일명**: `YYYY-MM-DD_retrospective.md`
- **헤더 필수 항목**:
    - 작성일자 (YYYY-MM-DD)
    - 시간 (HH:MM)
    - 주요 키워드 (#태그)

## 3. 작성 워크플로우 (6단계)
1. **[수신]**: 대표님의 회고 원문 및 "회고록 작성이야" 지시 확인.
2. **[분석]**: `syn-retro-analyzer` 스킬을 사용하여 3대 관점(강화/개선/전략) 분석.
3. **[이해보고서]**: 원문 + 분석 내용 초안 보고 및 피드백 요청.
4. **[피드백 반영]**: 내용 수정 및 보완.
5. **[최종 생성]**: 확정된 내용으로 `SYN_회고/` 폴더에 파일 생성.
6. **[깃허브 동기화]**: Push 완료 후 대표님께 깃허브 링크 제공.

## 4. 분석 필수 요소 (3-View)
- **🛡️ 강화점**: 현재 가장 잘 작동하고 있는 시스템이나 습관.
- **🛠️ 개선점**: 기술적 한계(API, 모델 설정 등)나 운영상 미흡한 점.
- **💡 차기 전략**: 개선점을 해결하기 위한 구체적인 액션 아이템 제언.

## 5. 아카이빙 관리
- 모든 회고록은 생성 즉시 **📚 아키비스트**에 의해 관리됩니다.
- 회고록 리스트는 `SYN_회고/` 내의 인덱스 파일(준비 중)을 통해 추적됩니다.
