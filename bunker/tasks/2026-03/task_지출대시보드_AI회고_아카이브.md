## 지시 원문

> 대시보드 제작 / 월별, 주별, 일간 사용 내역 확인과 등록된 지출로 이번 주, 이번달 지출 내역 예상하기
> - AI 기능으로 이번달 재무 회고하기 지출내역을 토대로 계획대로 진행되었는지, 과다지출, 지출을 잘 줄였는지 확인

## 5대 기능 블록 상세 설계

### 블록 1: 대시보드

```
┌─────────────────────────────────────────────────┐
│  📊 재무 대시보드                    [일간|주간|월간]  │
├────────────┬────────────┬────────────┬──────────┤
│ 💰 이번 달   │ 📅 이번 주   │ 📈 예상 지출  │ 🎯 계획   │
│ 총 지출     │ 총 지출     │ (등록된 미래) │ 달성률   │
│ ₩2,340,000 │ ₩580,000  │ ₩1,200,000 │ 78%     │
├────────────┴────────────┴────────────┴──────────┤
│  카테고리별 지출 비중 (도넛 차트)                      │
│  [소모품비 35%] [식대 20%] [교통비 15%] ...          │
├─────────────────────────────────────────────────┤
│  일별 지출 추이 (막대 차트) — 최근 30일               │
│  ▓▓▓ ▓▓ ▓▓▓▓ ▓ ▓▓▓ ▓▓ ▓▓▓▓ ▓▓▓              │
├─────────────────────────────────────────────────┤
│  최근 지출 내역 (테이블 — 최근 10건)                  │
│  날짜 | 항목 | 카테고리 | 금액 | 상태                 │
└─────────────────────────────────────────────────┘
```

**주요 로직:**
- 일간: 오늘 등록된 / 오늘 집행된 지출 합산
- 주간: 이번 주 월~일 범위의 지출 합산
- 월간: 이번 달 1일~말일 범위의 지출 합산
- 예상 지출: 상태가 `계획됨`이고 지출예정일이 이번 주/이번 달에 해당하는 건의 예상금액 합산
- 계획 달성률: `집행완료 건수 / 전체 건수 × 100`

### 블록 2: AI 재무 회고

```
┌─────────────────────────────────────────────────┐
│  🤖 AI 재무 회고                    [이번 달 회고]   │
├─────────────────────────────────────────────────┤
│  📋 회고 대상 기간: 2026년 3월                       │
│  총 계획: 15건 / 집행완료: 12건 / 취소: 3건           │
│  계획 총액: ₩5,000,000 / 실제 총액: ₩4,800,000     │
├─────────────────────────────────────────────────┤
│  🧠 Claude AI 분석 결과                             │
│  ─────────────────────────                       │
│  ■ 종합 평가: 계획 대비 96% 집행 — 양호               │
│  ■ 과다지출 항목: 접대비 (계획 대비 +45%)             │
│  ■ 절감 성공 항목: 소모품비 (계획 대비 -20%)           │
│  ■ 개선 제안: 접대비 사전 한도 설정 권장               │
│  ■ 다음 달 예측: 교육비 시즌으로 +15% 예상            │
├─────────────────────────────────────────────────┤
│  [📥 회고 저장]  [📄 PDF 내보내기]                    │
└─────────────────────────────────────────────────┘
```

**AI 프롬프트 설계:**
- 입력: 해당 월의 전체 지출 데이터 (계획 vs 실제)
- 분석 항목: 집행률, 카테고리별 과다/절감, 전월 대비 변화, 다음 달 제안
- 출력: 구조화된 JSON → UI에 렌더링
- 저장: `AI회고_기록` 시트에 기록

### 블록 5: 아카이브 + 분기 회고

```
┌─────────────────────────────────────────────────┐
│  📦 아카이브 관리                                   │
├─────────────────────────────────────────────────┤
│  월간 아카이브                                      │
│  ┌─────────────────────────────────────────┐    │
│  │ 2026년 2월 — ✅ 아카이브 완료 (15건)       │    │
│  │ 2026년 1월 — ✅ 아카이브 완료 (12건)       │    │
│  │ 2025년 12월 — ✅ 분기 이관 완료 (Q4)       │    │
│  └─────────────────────────────────────────┘    │
│                                                   │
│  현재 월 (2026년 3월) — 🔄 진행 중 (8건)            │
│  [AI 회고 실행] → 회고 완료 후 [아카이브 실행] 활성화  │
├─────────────────────────────────────────────────┤
│  분기 회고                                         │
│  ┌─────────────────────────────────────────┐    │
│  │ 📊 2026 Q1 회고 (1~3월)       [회고 시작] │    │
│  │ 📊 2025 Q4 회고 (10~12월)  ✅ 완료       │    │
│  └─────────────────────────────────────────┘    │
│                                                   │
│  분기 회고 시 Claude AI가 분석:                      │
│  - 분기 총 지출 추이                                │
│  - 카테고리별 변화 패턴                              │
│  - 프로젝트별 예산 소진율                            │
│  - 다음 분기 예산 제안                               │
└─────────────────────────────────────────────────┘
```

**아카이브 플로우:**
1. 월말 → AI 회고 실행 (블록 2)
2. 회고 저장 완료 → 아카이브 버튼 활성화
3. 아카이브 실행 → 해당 월 데이터를 `archive` 탭으로 이동 + `아카이브상태` = `archived`
4. 분기 말(3·6·9·12월) → 분기 회고 버튼 활성화
5. 분기 회고 완료 → `archive` 탭 데이터를 별도 스프레드시트(`지출아카이브_YYYY_QN`)로 이관

```
                              │
                              └─→ Phase 4 (프론트엔드)
                                    T-401 → T-402~T-407
                              │
                    ┌─────────┴─────────────────────┐
                    ▼                               ▼
              QA 체크포인트                    QA 체크포인트
              T-501a (P1~2 API)              T-501b (P3 AI)
                    │                               │
                    └─────────┬─────────────────────┘
                              ▼
                        T-501c (P4 통합 QA)
```

**병렬 가능 구간:**
- T-101, T-103, T-104는 서로 독립 → 동시 진행 가능
- Phase 2와 Phase 3는 Phase 1 완료 후 동시 진행 가능
- Phase 4의 T-401(탭 구조)은 Phase 2·3과 병렬 진행 가능
- **T-501a와 T-501b는 각각 Phase 1~2, Phase 3 완료 후 병렬 QA 가능**
- T-501c는 Phase 4 + T-501a/b 모두 PASS 후 실행
