# 📋 [실행 계획서] 출근 관리 자동화 시스템 (Attendance Automation)

## 프로젝트 개요
- **요청자**: 용남 대표
- **요청 유형**: 신규 백오피스 자동화 (기구축 프론트엔드 연동)
- **목적**: 주디 에이전트 채팅을 통한 간편한 출근 기록 및 근태 데이터 자산화
- **범위**: 데이터 시트 설계, 백엔드 API 구현, 사용자 매뉴얼 제작
- **긴급도**: 🟡 보통 (프론트엔드 선반영 상태)

## 현황 분석
- **AS-IS**: `judy_workspace.html`에 '출근' 키워드 감지 및 `handleCheckIn` 호출 로직이 추가되었으나, 백엔드(`handleCheckIn`) 함수가 부재하여 실행 시 오류 발생 예상.
- **TO-BE**: `handleCheckIn` 호출 시 Google Sheet의 '근태관리' 시트에 [날짜, 시간, 이름, 상태(출근)]가 기록되고, 주디가 "출근 처리가 완료되었습니다!"라는 응답을 반환함.

## 태스크 상세

### T-101: 근태 관리 데이터 시트 설계 (정DA)
- **입력**: 근태 기록에 필요한 필드 (일자, 시각, 이름, 구분)
- **산출물**: '근태관리' Google Sheet (또는 기존 관리 시트 내 새 탭)
- **완료기준**: 데이터 딕셔너리 포함, 중복 기록 방지 로직을 위한 키(Key) 설계 포함.

### T-102: `handleCheckIn` 백엔드 구현 (🤖 자비스 팀)
- **입력**: `userName`
- **산출물**: `src/gas/attendance.gs` (신규 파일)
- **완료기준**: `PropertiesService` 또는 시트 연동을 통해 기록 성공 시 성공 메시지 반환.

### T-103: 출근 시스템 사용자 가이드 작성 (박DC)
- **입력**: 구현된 기능 스펙
- **산출물**: `manuals/08_attendance_guide.md`
- **완료기준**: 꼼꼼이 팀의 매뉴얼 표준(v2.2) 준수, 인덱스 등록.

## 리스크 및 주의사항
- **비용**: API 호출 없이 `google.script.run`을 사용하므로 추가 API 비용은 발생하지 않음.
- **예외**: 동일 날짜 중복 출근 시 "이미 출근 처리되었습니다" 등의 방어 로직 필요.
