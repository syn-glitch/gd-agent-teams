## 【원본 지시】
> **새 업무 등록 기능 개선 및 신규 기능 추가**
> - 현재 : 프로젝트는 구글시트에서 등록하도록 한정 되어 있음
> - 변경 : 주디워크스페이스 모달창에서 등록 가능하게 변경해줘, 그러면 시트도 자동 연동
> - 현재 : 마감일만 등록 가능 
> - 변경 : 마감일을 시작일도 등록할 수 있도록해줘, 내가 시작일을 미리 지정하면 시작 3일전에 주디가 담당자에게 슬랙으로 알림이 올 수 있도록해줘
> - "신규 담당자 추가" : 새 업무를 만들어서 담당자를 지정하면 해당 담당자 업무에 보이고, 담당자가 수락을하면 업무를 신청한 사람과 내 업무 현황과 칸반에서 진행상황을 같이 볼 수 있도록 해줘
> - "메모" : 업무 카드에 메모창을 만들어서 업무에 메모를 남기면서 히스토리를 볼 수 있도록해줘

## 【의도 해석】
대표님께서는 현재 주디 워크스페이스의 업무 관리 시스템을 **"시트 의존성 탈피"**와 **"협업 및 이력 관리 강화"** 방향으로 고도화하고자 하십니다. 특히 단순한 업무 기록을 넘어, 담당자 배정부터 수락, 그리고 업무 진행 중 발생하는 커뮤니케이션(메모)까지 시스템 내에서 완결되기를 원하시는 것으로 파악됩니다.

## 【작업 범위】
1.  **프로젝트 직접 등록 (UI/UX)**:
    *   기존 '새 업무 등록' 모달 내 '프로젝트 선택' 셀렉트 박스에 '신규 프로젝트 추가' 기능 삽입.
    *   신규 프로젝트 입력 시 `Projects` 시트에 자동 기록 및 즉시 반영.
2.  **시작일(Start Date) 필드 추가 및 알림**:
    *   `Tasks` 시트에 '시작일' 컬럼 추가.
    *   모달창에 시작일 입력 필드 추가.
    *   시작 3일 전 담당자 대상 자동 슬랙 알림 트리거(GAS Trigger) 구현.
3.  **담당자 배정 및 수락 워크플로우**:
    *   업무 생성 시 `Status`를 '배정됨' 또는 '수락대기' 상태로 설정.
    *   담당자가 자신의 워크스페이스에서 해당 업무를 '수락'할 수 있는 버튼 구현.
    *   수락 후 신청자와 담당자 양쪽 모두에게 공유된 상태(진행중)로 표시.
4.  **업무별 히스토리 메모**:
    *   업무 상세/카드 클릭 시 메모를 입력하고 조회할 수 있는 타임라인형 메모창 구현.
    *   메모 데이터 저장 공간(신규 시트 또는 기존 시트 확장) 설계.

## 【산출물】
- **실행 계획서**: 각 팀(자비스, 정DA, 박DC 등)에 배분될 구체적인 태스크를 포함한 문서.
- **백엔드(GAS) 업데이트**: 시트 구조 변경 및 알림 로직, 수락 처리 API.
- **프론트엔드(HTML/CSS/JS) 업데이트**: 모달 UI 변경, 수락 버튼 추가, 참고사항 컴포넌트 구현.
- **사용자 매뉴얼**: 꼼꼼이 문서팀을 통한 가이드 업데이트.
