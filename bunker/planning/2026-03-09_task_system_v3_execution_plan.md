# 📋 [실행 계획서] 주디 워크스페이스 업무 관리 고도화 (Task System v3.1)

## 프로젝트 개요
- **목적**: 주디 워크스페이스 내에서 프로젝트 등록, 업무 배정/수락, 시작일 알림, 업무 메모 기능을 통합하여 협업 완결성 강화.
- **주요 변경점**:
    - 시트 접속 없이 주디 내에서 신규 프로젝트 등록 가능 (코드 자동 생성).
    - '시작일' 필드 도입 및 시작 3일 전 슬랙 자동 알림.
    - 업무 배정 -> 담당자 수락 -> 진행 이행 워크플로우 도입.
    - 업무별 텍스트 참고사항 및 외부 링크(구글 워크스페이스) 연동 히스토리 기능.

## 사용자 피드백 반영 사항
- ✅ **프로젝트 코드**: 수동 입력 대신 시스템에서 **자동 생성**.
- ✅ **알림 기준**: 주말/공휴일 관계없이 **단순 날짜 기준** (시작 3일 전).
- ✅ **참고사항/링크**: 텍스트 위주로 관리하되, 구글 드라이브/문서 링크 입력 시 **클릭 시 바로 이동**하도록 매직 링크 처리. (XSS 및 수식 주입 방지 로직 포함)
- ✅ **데이터 저장**: 하위 호환성을 위해 **기존 시트 맨 뒤에 열 추가** 및 신규 시트(탭) 생성. **LockService**를 통한 동시성 제어 필수 적용.

## Proposed Changes

### 1. 데이터 구조 설계 (Data Schema)

#### [MODIFY] `Tasks` 시트 (열 추가 - **맨 뒤에 추가하여 기존 인덱스 보존**)
- **J열 (신규)**: `Start_Date` (시작일, yyyy-mm-dd)
- **K열 (신규)**: `Created_By` (업무 신청자 ID/이름)
- **Status (C열) 값 확장**: `수락대기`, `진행중`, `완료`, `보유`, `삭제`

#### [NEW] `Task_References` 시트 (신규 탭)
- 업무별 히스토리를 저장하는 전용 공간.
- 컬럼 구성: `Ref_ID`, `Task_ID`, `Author`, `Content` (Text/Link), `Timestamp`

### 2. 컴포넌트별 구현 계획

#### [A] 주디 워크스페이스 (Frontend)
- **프로젝트 등록**: '새 업무 등록' 모달 내 프로젝트 선택창 옆에 [+] 버튼 추가 -> **인라인 입력창 전환** (모바일 배려).
- **수락 워크플로우**: '내 업무' 탭에서 `수락대기` 상태인 업무에 [✅ 수락] 버튼 노출. (본인이 담당자인 경우에만 활성화)
- **업무 참고사항**: 업무 카드 상세 보기 내 타임라인 UI 추가. 링크 포함 시 `<a>` 태그로 변환하되 **Sanitize 처리** 필수.

#### [B] 백엔드 로직 (GAS - 자비스 Ada 담당)
- `createProject(name)`: **LockService(ScriptLock)** 적용하여 중복 생성 방지 및 고유 코드 부여.
- `updateTaskStatus(taskId, status)`: **담당자 본인 확인** 후 `Status` 업데이트 및 히스토리 기록.
- `addTimelineReference(taskId, content)`: **XSS 및 시트 수식 주입(=) 방지 필터링** 후 참고사항 기록.
- **업무 조회**: `(Assignee == User) OR (Created_By == User)` 조건으로 필터링하여 신청자와 담당자 모두에게 노출.

#### [C] 알림 엔진 (Task Trigger)
- 매일 오전 9시 실행되는 트리거.
- `Start_Date` 기준으로 (오늘 + 3일) 인 업무 중 담당자에게 슬랙 멘션 알림 발송.
