#### 이슈 1: Slack 채널 ID 하드코딩 계획

**현재 계획 (STEP 1-2)**:

- Slack 채널 ID를 코드에 하드코딩하는 것은 **보안 위험**

- GitHub 등 외부 저장소에 실수로 업로드 시 채널 ID 노출

**영향 범위**:
- 관리자 채널 보안 침해 위험

#### 이슈 6: 관리자 알림 메시지 타임스탬프 형식 미정의

**STEP 1-4 코드 예시**:
```javascript
시간: ${timestamp}
```

**문제점**:
- `timestamp` 형식이 정의되지 않음 (ISO 8601? Unix time? 한국어 포맷?)
- 관리자가 보기 편한 형식 필요
- 예: `1709361045` (Unix) vs `2026-03-02 15:30:45 (KST)` (한국어)

**개선 방안**:
```javascript
function formatKoreanDateTime(timestamp) {
  const date = new Date(timestamp);
  const year = date.getFullYear();
  const month = String(date.getMonth() + 1).padStart(2, '0');
  const day = String(date.getDate()).padStart(2, '0');
  const hours = String(date.getHours()).padStart(2, '0');
  const minutes = String(date.getMinutes()).padStart(2, '0');
  const seconds = String(date.getSeconds()).padStart(2, '0');

  return `${year}-${month}-${day} ${hours}:${minutes}:${seconds} (KST)`;
}

// 사용
시간: ${formatKoreanDateTime(timestamp)}
// 출력: 2026-03-02 15:30:45 (KST)
```
