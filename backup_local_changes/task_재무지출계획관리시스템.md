## 지시 원문

> 송팀장, 내가 말한거 기획과 분석해줘
> - 회사에 팀원이 지출계획을 미리세워, 지출 목록, 지출날짜, 가격비교 등 재무 지출에 관련한 일정을 미리 세운다.
> - 팀원이 세운 일정에 맞게 재무 에이전트가 작동해서 미리 알림을 주고, 상관(팀장, 대표)에게 계획에 대해서 노티를 준다.
> - 마지막 하나의 재무 과정이 끝나면 실제 계획과 실제 지출을 비교하고, 실제 지출이 낮아지거나 높아진 이유에 대해서 보고하고 에이전트가 요약해준다.

- **핵심 요청**: 기존 캐시플로우 봇(경비 등록) 시스템에 **"지출 계획 → 실행 관리 → 사후 분석"** 3단계 사이클을 확장하는 시스템의 기획·분석
- **작업 범위**:

  - ❌ 제외: 실제 코드 개발 (→ 자비스 팀), UI/UX 디자인 상세 (→ 최AR), QA (→ 김감사 QA팀)
- **완료 기준**: 대표님이 검토·승인할 수 있는 기획서(PRD)가 마크다운으로 완성되고, 개발 위임용 텍스트 사양이 포함된 상태

## 기술 검토: 온라인 최저가 자동 검색

> 대표님 질문: "온라인 최저가 검색이 에이전트가 가능해? 실제 기술적 방법을 구체적으로 설명해줘"

### 기술적 방법 3가지

| 방법 | 설명 | 난이도 | 비용 | 정확도 |
|------|------|--------|------|--------|
| **A. 네이버 쇼핑 API** | 네이버 쇼핑 검색 API로 상품명 검색 → 최저가 반환 | 🟢 낮음 | 무료 (하루 25,000회) | 높음 (국내 쇼핑) |
| **B. 구글 Shopping API** | Google Custom Search API + Shopping 탭 검색 | 🟡 중간 | 유료 ($5/1,000건) | 높음 (글로벌) |
| **C. 웹 스크래핑 (MCP)** | MCP 브라우저 도구로 쇼핑몰 직접 조회 | 🔴 높음 | 무료 | 불안정 (사이트 변경 시 깨짐) |

### 방법 A 상세: 네이버 쇼핑 API (권장)

```
[구현 흐름]
1. 네이버 개발자센터에서 애플리케이션 등록 (Client ID + Secret 발급)
2. GAS에서 네이버 쇼핑 검색 API 호출:
   GET https://openapi.naver.com/v1/search/shop.json?query={상품명}&display=5&sort=asc
3. 응답에서 최저가(lprice), 상품명(title), 쇼핑몰(mallName), 링크(link) 추출
4. 팀원이 입력한 금액과 API 최저가를 비교
5. 결과를 Slack DM 또는 시트에 자동 기록

[실제 응답 예시]
{
  "items": [
    { "title": "A4용지 2500매", "lprice": "12900", "mallName": "쿠팡", "link": "..." },
    { "title": "A4용지 2500매", "lprice": "13500", "mallName": "G마켓", "link": "..." }
  ]
}

[한계점]
- 기업 대량 구매가는 반영 불가 (소매 최저가 기준)
- B2B 물품(특수 장비, 맞춤 제작)은 검색 불가
- 검색 키워드 품질에 따라 정확도가 달라짐
```
