## 1. 검수 배경
모바일 앱 스타일의 사용성을 강화하기 위해 대시보드 레이아웃, 하단 네비게이션, 메모 환경의 UI/UX를 개선합니다.

## 2. 검수 대상 (Deliverables)
- `src/frontend/judy_workspace.html`
- `src/frontend/judy_note.html`

## 3. 핵심 QA 시나리오 (UX 중심)

### SC-01: 대시보드 업무 카드 중심 레이아웃
- **검수 내용**: 대시보드 상단의 상황판(Summary Bar)이 모바일에서 세로 길이를 과도하게 차지하지 않는지 확인.
- **기대 결과**: 상황판 크기가 줄어들어 하단의 '진행 중 업무' 리스트가 한눈에 더 많이 노출되어야 함.

### SC-02: 하단 네비게이션 버튼 사용성
- **검수 내용**: 하단 탭바(노트, 업무, 칸반, 달력)의 버튼 크기 및 터치 영역 확인.
- **기대 결과**: 실제 핸드폰 작동 시 버튼이 충분히 커서 오작동 없이 터치가 용이해야 함 (터치 타겟 56px 이상 권장).

### SC-03: 메모 햄버거 메뉴 위치
- **검수 내용**: 메모 환경의 메뉴(☰) 버튼 위치 확인.
- **기대 결과**: 왼쪽 하단에서 **오른쪽 상단**으로 이동하여 한 손 조작 및 시각적 균형이 맞아야 함.

### SC-04: 메모 에디터 툴바 겹침 현상
- **검수 내용**: 텍스트 입력 시 나타나는 포맷 툴바와 입력 텍스트의 겹침 확인.
- **기대 결과**: 첫 줄 작성 시 에디터 버튼들이 텍스트를 가리지 않아야 함.

### SC-05: 메모 저장 버튼 시인성
- **검수 내용**: 저장 버튼과 하단 탭바의 배치 관계 확인.
- **기대 결과**: 일반 저장 버튼이 하단 탭바에 가려지지 않고, 탭바 바로 위쪽 적절한 위치에 노출되어야 함.

## 4. 심각도 분류 기준
- **CRITICAL**: 버튼 클릭 불가, 레이아웃 붕괴로 인한 기능 사용 불가.
- **MAJOR**: 디자인 가이드 미준수, 터치 영역 협소로 인한 사용성 저하.
- **MINOR**: 단순 오타, 색상 실효성 부족 등 미세한 개선 사항.

## 5. 검수 요청 사항
- 김감사 PO님은 위 시나리오를 바탕으로 UX검증관에게 검수를 배분해 주시기 바랍니다.
- 특히 **'모바일 실기기 환경'** 테스트 시뮬레이션을 중점적으로 진행 부탁드립니다.
