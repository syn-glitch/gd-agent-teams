## 🔍 영역별 QA 상세

### 1. 기능 (🔍 테스터) — 75점
- **[MAJOR] 인덱스 참조 하드코딩 리스크**: `Tasks` 시트 중간에 `Start_Date`를 삽입할 경우, 기존 `ai_chat.gs` 등에서 `row[7]`과 같이 하드코딩된 인덱스를 사용하는 로직이 모두 파괴될 가능성이 매우 높음. 
    *   *개선방안*: 열 삽입 대신 **맨 뒤에 열을 추가**하거나, 코드 내에서 **헤더명을 기반으로 인덱스를 동적으로 찾는 유틸리티** 도입 필수.
- **[MINOR] 공유 범위 모호**: '담당자가 수락하면 신청자와 함께 공유'한다는 로직에서, 필터링 기준이 명확하지 않음. `Created_By` 필드와 `Assignee` 필드의 OR 조건 처리가 필요함.

### 2. 보안 (🛡️ 보안감사관) — 60점
- **[MAJOR] 동시성 제어(LockService) 부재**: 프로젝트 자동 생성 및 상태 변경 시 여러 사용자가 동시에 접근할 경우 데이터가 덮어씌워질 위험이 있으나, 계획서에 `LockService` 적용에 대한 구체적 언급이 부족함.
- **[MAJOR] 입력값 검증(Injection) 누락**: 참고사항(구글 링크 등) 입력 시 스크립트 주입이나 시트 수식 주입(`=IMAGE(...)` 등)에 대한 필터링 계획이 누락됨.
- **[MINOR] 권한 체크**: 담당자가 아닌 타인이 업무를 '수락'하거나 참고사항을 수정하는 것을 방지하는 `Session.getActiveUser()` 기반의 서버 측 검증 로직이 설계에 포함되어야 함.

### 3. UX (🎨 UX검증관) — 85점
- **[MINOR] 모달 복잡도**: 프로젝트 신규 등록 시 현재 모달 내에서 또 다른 입력 레이어가 겹치게(Overlap) 되면 모바일에서 조작이 어려울 수 있음. 인라인 입력 방식으로 전환 권고.
- **[MINOR] 매직 링크 시인성**: 참고사항 내 링크가 일반 텍스트와 구분되도록 별도 아이콘(🔗) 또는 컬러링 가이드 필요.

## 🛠️ 수정 필요 항목 (Action Items)
1. **[보안/기능]** `Tasks` 시트 열 삽입 대신 **맨 뒤 추가**로 변경하고, `LockService` 적용 로직을 API 설계에 명시할 것.
2. **[보안]** 참고사항 입력 필드에 대한 **XSS 및 시트 수식 주입 방지 로직**을 자비스 Ada 태스크에 추가할 것.
3. **[기능]** 업무 조회 쿼리 시 `(Assignee == User) OR (Created_By == User)` 조건으로 조회하도록 로직 구체화.
4. **[UX]** 프로젝트 신규 등록 UI를 '입력창 전환' 또는 '인라인 추가' 방식으로 개선안 제시.
